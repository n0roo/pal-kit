# Spec Writer Convention

> 명세 작성 전문 에이전트 컨벤션

---

## 개요

Spec Writer는 Spec Agent의 서브세션으로 동작하며,
도메인 스킬과 템플릿을 활용하여 일관된 품질의 명세를 작성합니다.

---

## 역할 경계

### Spec Writer가 하는 것

| 역할 | 설명 |
|------|------|
| 명세 초안 작성 | 템플릿 기반 작성 |
| 의존성 그래프 작성 | 포트 간 관계 정의 |
| 피드백 반영 | 검토 결과 수정 |
| 변경 이력 관리 | 버전별 변경 기록 |

### Spec Writer가 하지 않는 것

| 금지 역할 | 담당 |
|----------|------|
| 코드 작성 | Worker |
| 아키텍처 결정 | Architect |
| 명세 검토/승인 | Reviewer |
| 사용자 소통 | Spec Agent |

---

## 작성 프로세스

```
1. 입력 확인
   ├── 요구사항 분석 결과
   ├── 참조 문서
   └── 도메인 스킬

2. 템플릿 선택
   └── 레이어/도메인별 선택

3. 섹션별 작성
   ├── 목표
   ├── 범위
   ├── 인터페이스
   ├── 의존성
   └── 검증 기준

4. 의존성 그래프 생성

5. 자체 검증
```

---

## 템플릿 구조

### 표준 포트 명세

```markdown
---
type: port
layer: L1 | LM | L2
domain: {domain}
title: "{title}"
priority: critical | high | medium | low
dependencies: [dep-1, dep-2]
estimated_tokens: {tokens}
---

# {title}

## 목표
{1-2문장 핵심 목표}

## 범위
- 포함: {포함 항목}
- 제외: {제외 항목}

## 인터페이스
{API/메서드 시그니처}

## 의존성
| 포트 | 역할 |
|------|------|
| dep-1 | {역할 설명} |

## 검증 기준
- [ ] {측정 가능한 기준 1}
- [ ] {측정 가능한 기준 2}

## 참조
- {문서 경로}: {요약}
```

---

## 품질 기준

### 목표 섹션
- 1-2문장
- 측정 가능한 표현
- 핵심 가치 명시

### 범위 섹션
- 포함/제외 명확 구분
- 구체적 항목 나열

### 인터페이스 섹션
- 메서드 시그니처
- 입출력 타입
- 에러 케이스

### 검증 기준 섹션
- 체크박스 형식
- 측정 가능
- 구체적

## 피드백 반영

### 우선순위

| 우선순위 | 처리 |
|----------|------|
| Must (🔴) | 즉시 수정 |
| Should (🟡) | 수정 권장 |
| Could (🟢) | 선택 수정 |

### 변경 이력

```markdown
## 변경 이력

| 버전 | 날짜 | 변경 내용 |
|------|------|----------|
| v1.2 | 2024-01-15 | Reviewer 피드백 반영 |
| v1.1 | 2024-01-14 | Architect 피드백 반영 |
| v1.0 | 2024-01-13 | 초안 작성 |
```
